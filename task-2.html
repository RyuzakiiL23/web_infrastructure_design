Explanation of the given infrastructures

  • Firewalls:

    Firewalls are security devices or software applications designed to monitor and control incoming and outgoing network traffic based on predetermined security rules. They act as a barrier between a trusted internal network and untrusted external networks, helping to prevent unauthorized access, data breaches, and other cyber threats. Firewalls can block or allow traffic based on factors like IP addresses, port numbers, and application protocols.

  • Traffic served over HTTPS:

    HTTPS (Hypertext Transfer Protocol Secure) is used to serve traffic over an encrypted connection between a client (user's browser) and a server. This encryption ensures that data transmitted between the client and server remains confidential and cannot be easily intercepted or tampered with by malicious actors. HTTPS uses SSL/TLS protocols to establish a secure communication channel, providing authentication, data integrity, and encryption.

  • Monitoring:

    Monitoring involves observing and tracking the performance, health, and availability of various components within an infrastructure. It helps ensure that systems are operating optimally, identifies potential issues or anomalies, and aids in maintaining overall system reliability. Monitoring can involve tracking metrics like CPU usage, memory utilization, network traffic, response times, error rates, and more.

  • Collecting Data in Monitoring:

    Monitoring tools collect data through various means, such as agents installed on servers, polling of network devices, analyzing log files, and using APIs provided by different components. The collected data is then processed, analyzed, and presented in the form of visualizations, dashboards, and reports for administrators to make informed decisions and take actions as needed.

  • Monitoring Web Server QPS:

    To monitor the web server's Query Per Second (QPS), you can use monitoring tools like Prometheus, New Relic, or Datadog. These tools often provide built-in metrics for tracking QPS. You would need to configure the monitoring tool to gather data from the web server, which could involve setting up relevant plugins or agents and specifying the metrics you want to track. The monitoring tool will then continuously collect QPS data and display it on dashboards for analysis.

The issues with the given infrastructures

  • Terminating SSL at the Load Balancer Level:

    Terminating SSL at the load balancer exposes decrypted traffic within the internal network, potentially exposing sensitive data. SSL termination might be necessary for load balancing, but it could lead to data exposure if not managed properly.

  • Single MySQL Server Accepting Writes:

    Having only one MySQL server capable of accepting writes creates a single point of failure. If the server fails or becomes overwhelmed, it could lead to downtime and data loss. Replication and clustering techniques should be considered to enhance fault tolerance.

  • Uniformity of Servers:

    Using identical components for all servers (database, web server, application server) might lead to resource contention issues. Different components have varying resource requirements. Using the same server configuration for all components might not optimize performance and resource utilization.
